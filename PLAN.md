# PLAN.md — План реализации fib_bot (MVP → Production)

## 1) Подтвержденные входные требования

- Основная биржа: Hyperliquid.
- Резервная биржа: MEXC.
- Режим старта: paper trading по WebSocket + отдельный режим только сигналов.
- Исполнение в paper: по close свечи.
- Дефолтный таймфрейм: 5m.
- Таймфрейм должен настраиваться через Telegram-бота.
- Риск на сделку: максимум 2% депозита.
- Дневной лимит просадки: 10%, после чего авто-пауза до UTC 00:00.
- Отчет по позиции в Telegram: настраиваемый интервал, по умолчанию 1 час.
- Управление позицией из Telegram: разрешено изменение SL и TP, а также закрытие позиции.
- Режим доступа бота: настраиваемый (только уведомления / полный функционал).
- ML-фильтр: включается сразу в MVP, с возможностью анализа исторических данных.
- Whitelist активов: отложено в TODO.
- Расширенная новостная фильтрация: отложено в TODO, временный источник t.me/cryptoarsenal.

---

## 2) Цели MVP

- Запустить устойчивый контур signal-only и paper с контролем риска.
- Обеспечить переключение режима через Telegram без перезапуска кода.
- Реализовать 3 стратегии, regime-классификацию и ML-фильтр вероятности.
- Добавить пользовательский mini-backtest по выбранному активу (3000 свечей, выбранный ТФ) с отправкой отчета в Telegram.
- Получить воспроизводимые метрики: PF, DD, Winrate, Expectancy, число сделок.

## 4) Режимы работы и state machine

### Режимы

- `signal_only`: только сигналы и отчеты, без исполнения.
- `paper`: имитация сделок по close свечи.
- `live`: зарезервирован для следующего этапа после одобрения.

### Переходы состояния

- `running` → `paused` вручную из Telegram.
- Авто-переход `running` → `paused` при дневной просадке 10%.
- Авто-возврат `paused` → `running` только после UTC 00:00 и явного подтверждения.

## 5) Telegram-функционал MVP

### Базовые команды

- `/start` — мастер-профиль (режим, биржа, ТФ, риск, период отчета).
- `/mode` — переключение signal_only/paper.
- `/set_tf` — изменить рабочий таймфрейм.
- `/set_risk` — риск на сделку (ограничение максимум 2%).
- `/status` — текущий режим, риск, лимиты, health.
- `/positions` — текущие позиции с action-кнопками.
- `/set_sl` и `/set_tp` — изменение уровней позиции.
- `/close` — закрытие позиции.
- `/backtest` — mini-backtest по активу и ТФ.

### Уведомления

- Новый сигнал.
- Открытие/закрытие позиции.
- Периодический отчет по открытым позициям (дефолт 1 час, настраивается).
- Критические алерты (достигнут дневной лимит, ошибки биржи, деградация данных).

### Режим доступа

- `notify_only`: только уведомления, без торговых кнопок.
- `full_access`: полный контроль из Telegram.

## 6) Алгоритм торговли и риск

### Стратегический слой

- Strategy 1: Trend Pullback.
- Strategy 2: Volatility Breakout.
- Strategy 3: Liquidity Sweep Reversal.

### Regime + ML

- Regime classifier выбирает подходящую стратегию.
- ML probability filter включается сразу и фильтрует входы.
- Исторический анализ обязателен до активации актива в рабочем контуре.

### Риск-контур

- Max risk per trade = 2%.
- Max daily drawdown = 10%, затем блок новых сделок до UTC 00:00.
- Ограничение числа одновременных позиций.
- Контроль последовательности убыточных сделок.

## 7) Mini-backtest по запросу пользователя

### Сценарий

1. Пользователь выбирает актив и таймфрейм.
2. Система загружает 3000 свечей с выбранной биржи.
3. Выполняется тест стратегий + ML-фильтра.
4. В Telegram отправляется отчет.

### Состав отчета

- PF.
- Max DD.
- Winrate.
- Средний R:R.
- Expectancy.
- Число сделок.
- Краткий вывод: актив допущен/не допущен в торговый список.

- Команда запуска.

- Подготовка к следующему live-этапу.
